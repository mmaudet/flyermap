üß™ Test Cases

Test A: New Zone (Unassigned)

Steps:
  1. Click polygon tool in left toolbar
  2. Draw a new zone polygon on the map
  3. Enter a zone name when prompted
  4. Observe the zone color
Expected: Zone displays with red border (#ef4444) and light red fill (#fca5a5, opacity 0.2)

Test B: Assign First Team Member

Steps:
  1. Click on the zone to open editor dialog
  2. Select the first team member in the assignment dropdown
  3. Click "Enregistrer" to save
  4. Observe the zone color change
Expected: Zone immediately changes to magenta (#c30b82 border and fill, opacity 0.3)

Test C: Remove Assignment

Steps:
  1. Click on the magenta zone to open editor
  2. Deselect all team members (clear selection)
  3. Click "Enregistrer" to save
  4. Observe the zone color change
Expected: Zone returns to red (#ef4444 border, #fca5a5 fill)

Test D: Assign Second Team Member

Steps:
  1. Click on the zone to open editor
  2. Select the second team member in the list
  3. Click "Enregistrer" to save
  4. Observe the zone color
Expected: Zone changes to blue (#2563eb border and fill, opacity 0.3)

Test E: Multiple Assignments (First Takes Priority)

Steps:
  1. Click on the blue zone to open editor
  2. Hold Ctrl/Cmd and select multiple team members
  3. Ensure the second member (blue) is selected first in the list
  4. Click "Enregistrer" to save
  5. Observe the zone color
Expected: Zone displays color of the FIRST assigned member in the list (should remain blue #2563eb)

Test F: Persistence After Page Refresh

Steps:
  1. Ensure you have zones with various assignments (from tests above)
  2. Note the current colors of each zone
  3. Refresh the page (F5 or Cmd+R)
  4. Wait for the map to load
  5. Compare zone colors to pre-refresh state
Expected: All zone colors persist correctly:
  • Assigned zones display correct team member colors
  • Unassigned zones remain red
  • Colors match exactly as before refresh
